## Core Concept

Before spending Vercel/v0 credits or asking AI to generate code, the repo should be build-ready.

This means the product should already have:

- Product vision
- MVP scope
- UI spec
- Data loading plan
- Chat behavior spec
- Local state sample
- Component plan
- Build prompt
- Cost-safe plan

## Why This Matters

AI builders work better when the repo already explains what to build.

This reduces:

- Credit waste
- Wrong features
- Scope creep
- Rewrites
- Confusing output

## Build Readiness

The first prototype should be:

- Frontend-only
- Local state
- Mock chat
- No backend
- No database
- No real AI
- No paid APIs

## Key Lesson

Do not build too early.

A clear product backbone makes the first prototype cheaper and more aligned.
