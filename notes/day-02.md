# Day 2 Notes: Repo as AI Memory

## What I learned today

A repository can help AI tools work better when the project is structured clearly.

Instead of keeping all project information inside a long chat, I can store important context in files such as:

- README.md
- AGENTS.md
- PRD.md
- TASKS.md
- templates

## Why this saves tokens

AI tools waste more context when they need to read long conversations or guess what the project is.

A clear repo helps me tell the AI:

- which files to read
- which task to complete
- which files not to touch
- what the project goal is
- what constraints matter

## My new workflow

1. Store project goals in README.md
2. Store product details in PRD.md
3. Store small tasks in TASKS.md
4. Store AI assistant instructions in AGENTS.md
5. Ask AI to complete one task at a time

## Example token-saving instruction

Read AGENTS.md first.
Then read only the PRD.md and TASKS.md inside the relevant project folder.
Complete one task only.
Do not rewrite unrelated files.
