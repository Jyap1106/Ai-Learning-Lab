# Versioned Product Roadmap: Holiday Companion Bot

## Purpose

This file defines how the Holiday Companion Bot should be built in product versions.

The goal is to use the full GitHub repository as the product brain, but build the actual product in clear versions.

This avoids two problems:

1. Building too little and not proving the product.
2. Building too much and creating a fragile app.

## Product Vision

Holiday Companion Bot is an AI-assisted travel product that helps users build, follow, edit, save, and eventually share flexible travel itineraries.

The product should eventually support:

- Itinerary building
- Today and tomorrow trip guidance
- Chat-based trip assistance
- Itinerary editing
- Version history
- Saved itinerary state
- Sharing
- Future AI suggestions
- Future live integrations

## Build Philosophy

The product should be built in versions.

The full vision should be documented, but each build should focus only on the current version.

The build philosophy is:

```text
Think full product.
Build one version at a time.
Leave clean hooks for future versions.
